Rows Of B - the matrix calculator

Rows of B is an online, expression-based matrix calculator.

A Tutorial

Basics

Rows of B has a simple interface; at first, all you see is a single textbox. Try typing some basic math, such as 5 + 5 or 20*8 + 2, into that box (When you press enter, you should see the result of your input in a green box. Whenever we say “input into the calculator” or “input into the input box” we also intend for you to press enter). Try entering some invalid input as well, such as 1+ or %. You will see that an error appears below the input in place of the result box.

You have seen how scalar expressions work. Rows of B supports addition, subtraction, multiplication, division, and negation for scalar numbers. Note also that Rows of B operates entirely on the concept of fractions, not decimals. Thus, an input of 123 / 5 will appear in the results box as 123/5, and not as 24.6. You also cannot enter decimal numbers into the calculator. Note also that as of the latest version, Rows of B does not support exponents.

Variables

What about matrices? Another good question! Rows of B supports 52 variables: 26 for scalars and 26 for matrices. The scalar variables are all lowercase letters of the English alphabet and are preset to 0. Enter a or b into the calculator and you will get 0 as your result. However, the variable z is special: it is automatically set to the result of the last scalar calculation. Try it by evaluating an expression and then entering z.

The matrix variables are the uppercase letters of the English alphabet. Each matrix variable is preset to a 3x3 zero matrix. Just like with scalars, Z is a special matrix variable that is automatically set to the result of the last matrix calculation. Thus, if you enter D or Q or R into the calculator, you should see a 3x3 zero matrix in the result box.

Matrix Input

What if you want to set a variable’s value? Simply end your expression with -> s, where s is the name of the variable in which you want to store the expression’s result. For example, 3 + 2 -> a will store 5 in a. But what about matrices? How can they be set? Rows of B has a special syntax for matrix input. Enter the following into the calculator:

$$

You will see a blue box appear below the input box. Your cursor will be automatically placed within a box inside a 1x1 matrix. For every $$ you enter in your expression, a matrix input will appear in the blue box. Enter your (1,1) entry, then press TAB. Another column will be added to your matrix. Keep entering numbers and pressing TAB until your first row is complete. At that time, press ENTER, and a new row will start. Continue entering numbers there until you are satisfied. Any empty boxes will be converted to zero later, so don’t worry about skipping any. When you are finished defining the current matrix, press CTRL-ENTER. This will either move to the next matrix (if you wish to define more than one) or evaluate your expression.

Try entering $$ + $$ into the expression box. Then input two matrices into the blue box. Make sure they are the same size! Then, when you press CTRL-ENTER after completing the second matrix, the result of adding the two together will appear in the green box! That is, unless you’ve made a mistake. Also try entering 5*$$ or $$/6 -> A!

The $ notation comes in handy if you want to define a specific matrix variable and use it at the same time. Sure, you could simply do $$ -> F to set F to a custom matrix, but let us say that you were going to immediately input F * 3. It would be easier to skip the first step and instead enter $F * 3. This will prompt you for the value of F prior to evaluating the expression. After evaluation, F will still hold the value you entered.

Functions

Functions allow one to run algorithms to modify or create matrices. You can evaluate a function with the syntax name(argument1) or name(argument1, argument2). The arguments depend on the function. Rows of B currently includes the following functions:

  • ident(scalar <size>): Produces an identity matrix with size size.
  • aug(matrix <a>, matrix <b>): Augments a with b.
  • ref(matrix <m>): Produces m in a row echelon form.
  • rref(matrix <m>): Produces m in reduced row echelon form.
  • inv(matrix <m>): Inverts m if possible.

Functions can be used anywhere a matrix variable can be used. Here are some examples:

rref($$)
aug($A, B)
5*ident(3)
ident(v)/3
rref(inv($$))
